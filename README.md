## Problem Evaluation Framework

Every problem goes through this filter before we commit resources:

### Stage 1: Problem Statement
- **What's the pain?** (1 sentence)
- **Who is the user?** (specific persona)
- **How big is the pain?** (frequency √ó severity)
- **Are they paying to solve it now?** (existing spend)

### Stage 2: Economics
- **Revenue model?** (how do we get paid)
- **Unit economics?** (cost to serve vs revenue per user)
- **Market size?** (TAM/SAM/SOM)
- **Margins?** (bot labor = near-zero marginal cost)

### Stage 3: Feasibility & Distribution
- **Can we build it in 3 days?** (MVP scope)
- **Technical complexity?** (1-5 scale)
- **Distribution channel?** (how users find us)
- **Moat?** (why not just copy us)
